Asegurando APIs con OAuth2 y JWT en .NET

Por: Mtr. Alejandro Torres

Especialista de Ciberseguridad en SOFTIA.ES

Proteger los datos corporativos y controlar quién tiene acceso a los recursos expuestos por una API es uno de los pilares del desarrollo de software seguro. En el desarrollo moderno de aplicaciones distribuidas, la arquitectura sin estado (stateless) es la más recomendada, y la autenticación basada en **JSON Web Tokens (JWT)** junto al marco de autorización **OAuth2** constituye el estándar de la industria.

1. ¿Qué es un JSON Web Token (JWT)?

Un token JWT es un objeto JSON codificado en base64 que se envía en la cabecera `Authorization` de las peticiones HTTP mediante el esquema `Bearer`. Este token se compone de tres partes diferenciadas separadas por puntos:

  • Header (Cabecera): Especifica el tipo de token y el algoritmo de firma hash utilizado (ej. HMAC SHA256 o RSA).
  • Payload (Carga Útil): Contiene los "Claims" o declaraciones que representan los datos del usuario (roles, identificador, tiempo de expiración del token).
  • Signature (Firma): Permite al servidor web validar que el token es legítimo y que no ha sido alterado por un intermediario en el camino.

2. Configuración de JWT Bearer en ASP.NET Core 8

Configurar la autenticación JWT en C# requiere añadir y configurar el middleware correspondiente. A continuación, se presenta la sección clave de registro de servicios en el archivo `Program.cs`:

using Microsoft.AspNetCore.Authentication.JwtBearer;
using Microsoft.IdentityModel.Tokens;
using System.Text;

var builder = WebApplication.CreateBuilder(args);

// Registro de autenticación JWT
builder.Services.AddAuthentication(options =>
{
    options.DefaultAuthenticateScheme = JwtBearerDefaults.AuthenticationScheme;
    options.DefaultChallengeScheme = JwtBearerDefaults.AuthenticationScheme;
})
.AddJwtBearer(options =>
{
    options.TokenValidationParameters = new TokenValidationParameters
    {
        ValidateIssuer = true,
        ValidateAudience = true,
        ValidateLifetime = true,
        ValidateIssuerSigningKey = true,
        ValidIssuer = "https://identity.softia.es",
        ValidAudience = "https://api.softia.es",
        IssuerSigningKey = new SymmetricSecurityKey(Encoding.UTF8.GetBytes("ClaveSecretaSuperProtegidaDeSOFTIADe256Bits"))
    };
});

var app = builder.Build();
app.UseAuthentication(); // Activa el middleware de autenticación
app.UseAuthorization();  // Activa el middleware de autorización

3. Comparativa de Algoritmos de Firma para JWT

Elegir el algoritmo adecuado influye tanto en la seguridad como en la sobrecarga computacional del servidor. La siguiente tabla compara los algoritmos soportados en .NET:

Tabla 4: Análisis de algoritmos criptográficos para la firma de JSON Web Tokens en .NET.
Algoritmo Tipo de Clave Nivel de Seguridad Velocidad de Firma Complejidad de Manejo
HS256 (HMAC SHA-256) Simétrica (Secreta compartida) Bueno (Requiere claves largas) Ultra Rápido Baja
RS256 (RSA con SHA-256) Asimétrica (Pública / Privada) Muy Alto Moderado (Alto uso de CPU) Media (Rotación de certificados)
ES256 (ECDSA P-256 y SHA-256) Asimétrica (Curva Elíptica) Excelente (Seguridad fuerte con claves cortas) Rápido Alta
