Microservicios y APIs en entornos .NET

Por: Dra. Sofía Rivas

Líder de Arquitectura e Integración en SOFTIA.ES

En el diseño de sistemas distribuidos modernos, la transición de una arquitectura monolítica hacia una basada en **microservicios** es uno de los pasos más importantes para ganar escalabilidad y agilidad organizativa. Sin embargo, exponer directamente docenas de microservicios independientes al cliente final genera problemas de seguridad, CORS, acoplamiento y mantenimiento.

1. El rol del API Gateway

La solución de diseño recomendada por SOFTIA.ES es la implementación de una capa intermedia de enrutamiento conocida como **API Gateway**. Este componente centraliza los puntos de entrada públicos, realizando funciones de balanceo de carga, agregación de respuestas, terminación SSL y rate limiting.

En el ecosistema .NET actual, disponemos de dos herramientas muy potentes para construir un API Gateway a medida:

  • Ocelot: Una solución clásica de código abierto, muy personalizable mediante configuración JSON.
  • YARP (Yet Another Reverse Proxy): La solución modular y ultra rápida desarrollada directamente por Microsoft utilizando la infraestructura de enrutamiento integrada de Kestrel.

2. Configuración Simplificada de YARP

A continuación se describe la configuración típica de YARP en el archivo `appsettings.json` para enrutar peticiones hacia dos microservicios backend independientes (Clientes y Pedidos):

{
  "ReverseProxy": {
    "Routes": {
      "ruta-clientes": {
        "ClusterId": "cluster-clientes",
        "Match": { "Path": "/api/v1/clientes/{**catch-all}" }
      },
      "ruta-pedidos": {
        "ClusterId": "cluster-pedidos",
        "Match": { "Path": "/api/v1/pedidos/{**catch-all}" }
      }
    },
    "Clusters": {
      "cluster-clientes": {
        "Destinations": {
          "destino1": { "Address": "http://microservicio-clientes:8080" }
        }
      },
      "cluster-pedidos": {
        "Destinations": {
          "destino1": { "Address": "http://microservicio-pedidos:8080" }
        }
      }
    }
  }
}
